🔐 Proceso de Autenticación y Control de Acceso

Gestión del inicio de sesión, validación de credenciales, cambio de contraseña obligatorio y acceso al sistema SmartFlow según el rol del usuario.

Actor: Usuario Actor: Sistema Trigger: Manual BPMN 2.0
Pasos del proceso
1

Acceso a la plataforma

El usuario navega a la URL de SmartFlow y accede al formulario de inicio de sesión.

2

Ingresar credenciales

Se ingresan correo electrónico y contraseña. El correo es el identificador único en el sistema.

3

Validación en servidor

El servidor valida las credenciales contra la base de datos PostgreSQL. Devuelve JWT + refresh token si son correctas.

4

Verificar must_change_password

Si el admin creó la cuenta con contraseña temporal, el flag must_change_password obliga a cambiarla antes de continuar.

5

Cambio de contraseña (si aplica)

El usuario establece una contraseña personal. El sistema actualiza el flag y emite nuevos tokens.

6

Cargar Dashboard

Se carga el perfil completo del usuario (rol, negocio principal, permisos). Se redirige al Dashboard personalizado.

📁 Proceso de Gestión de Proyectos

Ciclo de vida completo de un proyecto: creación, planificación con épicas y sprints, ejecución mediante tablero Kanban/Backlog, seguimiento y cierre.

Actor: Líder de Proyecto Actor: Miembro del Equipo Actor: Admin Trigger: Manual
Líder/Admin
Miembro del equipo
Sistema automático
Pasos del proceso
1

Crear Proyecto

El líder/admin crea el proyecto definiendo nombre, negocio (CAS/BO), descripción y fecha objetivo.

2

Definir equipo

Se asignan miembros al proyecto. Cada miembro puede tener tareas asignadas dentro del proyecto.

3

Crear Épicas

Se definen épicas que agrupan funcionalidades o áreas de trabajo. Cada épica tiene estado, prioridad y responsable.

4

Planificar Sprint

Se crea un sprint con fechas de inicio/fin y objetivo. Las tareas se asignan al sprint desde el backlog.

5

Crear y asignar tareas

Las tareas se crean con prioridad, story points, fecha límite y se asignan a miembros del equipo.

6

Ejecutar tareas

El miembro mueve las tareas en el tablero Kanban: Pendiente → En Progreso → Revisión → Completada.

7

Monitorear avance

El líder revisa el tablero, backlog y métricas. Puede reasignar tareas o ajustar prioridades.

8

Cerrar Sprint / Proyecto

Al completar el sprint se revisan las tareas pendientes. Al completar el proyecto se archiva o elimina.

🚨 Proceso de Gestión de Incidentes

Reporte, investigación, resolución y cierre de incidentes operativos. Incluye escalamiento a Gestión de Demanda cuando el incidente supera la capacidad de resolución normal.

Actor: Reporter Actor: Responsable Actor: Líder Trigger: Incidente detectado
Pasos del proceso
1

Detectar y reportar incidente

Cualquier usuario reporta el incidente indicando título, descripción, severidad (Bajo/Medio/Alto/Crítico) y negocio afectado.

2

Asignación automática de número

El sistema genera un número único INC-XXXX y asigna estado "Abierto" automáticamente.

3

Asignar responsable

El líder asigna un responsable para investigar el incidente y lo mueve a "En Investigación".

4

Investigar y documentar

El responsable investiga, documenta la causa raíz y agrega comentarios al timeline del incidente.

5

Resolver incidente

Se documenta la solución aplicada, notas de resolución y se cambia el estado a "Resuelto".

6

Escalar a Demanda (si aplica)

Si el incidente requiere un cambio mayor, se escala creando una Gestión de Demanda vinculada al incidente.

7

Cerrar incidente

El líder verifica la solución y cierra el incidente. Queda en el historial con toda la trazabilidad.

🎙️ Proceso de Notas de Voz y Transcripción

Captura de audio, transcripción automática mediante IA (OpenAI Whisper → Groq → Web Speech), revisión y asignación de la nota a proyecto/tarea.

Actor: Usuario Actor: IA (Whisper) Trigger: Acción usuario
Pasos del proceso
1

Iniciar grabación

El usuario presiona el botón de micrófono (FAB flotante o módulo Notas de Voz). El navegador solicita permiso de micrófono.

2

Grabar audio

Se graba el audio usando la API MediaRecorder del navegador. El usuario puede ver la duración en tiempo real.

3

Transcripción OpenAI Whisper

El audio se envía al servidor que llama a OpenAI whisper-1 ($0.006/min). Si falla, se usa Groq como fallback gratuito.

4

Fallback Web Speech API

Si ambos servicios cloud fallan, se activa la Web Speech API del navegador para dictado en tiempo real (es-CO).

5

Revisar transcripción

El usuario revisa y edita el texto transcrito antes de guardar. Puede modificar el título generado automáticamente.

6

Asignar a proyecto/tarea

Opcionalmente se asocia la nota a un proyecto y/o tarea existente. Si no se asigna, queda en el Inbox.

7

Guardar nota

La nota se guarda con prioridad, fecha de vencimiento y estado. Aparece en la pestaña correspondiente (Inbox / Mis tareas / Delegué).

🤝 Proceso de Reuniones con IA (ARIA)

Creación de reunión, grabación con transcripción en tiempo real, generación de resumen con IA Gemini, actas de decisiones y acción items automáticos.

Actor: Organizador Actor: Participantes Actor: ARIA (IA) Trigger: Reunión programada
Pasos del proceso
1

Crear sesión de reunión

El organizador crea una reunión indicando tipo (interna, cliente, revisión), negocio y opcionalmente vinculando a un BP.

2

Compartir código de sesión

El sistema genera un código único (6 caracteres). Los participantes pueden unirse con ese código desde sus dispositivos.

3

Iniciar grabación

El organizador inicia la grabación. El audio se captura en chunks de 5 segundos y se envía al servidor automáticamente.

4

Transcripción en tiempo real

Cada chunk es transcrito por OpenAI Whisper → Groq → local. El texto aparece en pantalla en tiempo real con identificación de hablante.

5

Finalizar reunión

El organizador detiene la grabación y presiona "Finalizar". Se consolida la transcripción completa.

6

Resumen IA con Gemini

ARIA procesa la transcripción completa y genera: resumen ejecutivo, decisiones tomadas, action items y temas clave.

7

Revisar y usar actas

El equipo revisa el resumen generado, puede crear tareas directamente desde los action items y compartir las actas.

⚡ Proceso de Tareas Rápidas

Gestión de tareas operativas no asociadas a proyectos. Vinculadas a negocios (CAS/BO), con seguimiento de tiempo mediante Pomodoro y visibilidad para el líder.

Actor: Usuario Actor: Líder (vista) Trigger: Acción usuario / FAB
Pasos del proceso
1

Crear tarea rápida

Desde el FAB flotante o el módulo de Tareas Rápidas. Se define título, negocio, prioridad, tiempo estimado y fecha límite.

2

Asignar o delegar

La tarea puede ser propia o delegada a otro miembro del equipo. Si se delega, el destinatario es notificado.

3

Iniciar Pomodoro

El usuario activa el temporizador Pomodoro (25 min trabajo / 5 min descanso) directamente desde la tarjeta de la tarea.

4

Registrar tiempo

El tiempo trabajado se acumula automáticamente. Se puede ver el progreso vs. tiempo estimado en la barra de progreso.

5

Actualizar estado

El usuario puede cambiar el estado de la tarea (pendiente → en progreso → completada) en cualquier momento.

6

Vista del líder

El líder/admin ve todas las tareas rápidas agrupadas por negocio, con indicadores de vencidas y urgentes.

📋 Proceso de Gestión de Demandas

Ciclo de vida de solicitudes de cambio, nuevas funcionalidades o iniciativas estratégicas. Incluye análisis de situación, aprobación, ejecución y cierre formal.

Actor: Solicitante Actor: Líder / Directivo Actor: Equipo ejecutor Trigger: Necesidad identificada
Pasos del proceso
1

Crear demanda

El solicitante documenta la situación actual, situación deseada y justificación. Opcionalmente vinculada a un incidente escalado.

2

Análisis y revisión

El líder/directivo revisa la demanda, valida viabilidad, impacto y alineación estratégica.

3

Decisión: Aprobar / Rechazar

Se aprueba con recursos asignados o se rechaza documentando el motivo. El solicitante es notificado.

4

Planificación de ejecución

Si se aprueba, se asignan responsables, se definen hitos y se vincula a un proyecto si corresponde.

5

Ejecución y seguimiento

El equipo ejecuta la demanda. Se documentan avances, cambios de estado y actualizaciones periódicas.

6

Cierre y validación

El líder valida que la situación deseada fue alcanzada y cierra formalmente la demanda con acta de cierre.
